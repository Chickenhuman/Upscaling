## UI 방향

### 미연시 대화 화면

전형적인 VN 화면을 기본형으로 삼는다. 배경 위에 성인 캐릭터 전신 또는 반신이 떠 있고, 하단에는 큰 대화창이 있으며, 대화창에는 이름표와 선택지가 붙는다. 현재 구현처럼 합성 이미지를 잘라 쓰는 방식이 아니라 캐릭터별 PNG/WEBP 일러스트를 독립 레이어로 올린다.

필요 에셋:
- 캐릭터별 기본 전신 또는 반신 일러스트 5종.
- 캐릭터별 표정 차분 최소 3종.
- 하단 대화창 프레임.
- 이름표 프레임.
- 선택지 버튼 3상태: 기본, hover/focus, 선택됨.
- 상단 HUD 아이콘: 덱, 로그, 저장, 메뉴.

### 덱빌딩 전투 화면

카드가 손맛 있게 보이는 하단 손패 중심 UI로 간다. 중앙에는 적 의도와 체력, 하단에는 카드 5장, 좌우에는 드로우/버림 더미와 플레이어 상태를 둔다. 카드 사용 시 피해/회복/방어 숫자가 튀는 연출을 전제로 구성한다.

필요 에셋:
- 카드 프레임: 기본, 희귀, 전용, 비활성.
- 카드 패밀리 아이콘: 매력, 집중, 기지, 간식, 연계, 승부.
- 에너지 카운터.
- 턴 종료 버튼.
- 피해/회복/방어 팝업 숫자 스타일.
- 적 의도 아이콘: 공격, 방어, 버프, 디버프.
- 드로우/버림 더미 아이콘.

### 루트 맵 화면

지도는 단순 노드 버튼이 아니라 분기형 보드게임 경로처럼 보이게 한다. 각 주차마다 갈림길이 있고, 플레이어가 이번 회차의 리스크와 보상을 읽을 수 있어야 한다.

필요 에셋:
- 노드 아이콘: 전투, 이벤트, 휴식, 상점, 엘리트, 보스.
- 선택 가능/선택됨/잠김 노드 상태.
- 경로 라인과 진행 리본.
- 동료 호감도 패널.
- 유물/덱 요약 사이드 패널.

### ESC 메뉴 오버레이

배경을 흐리게 둔 상태에서 중앙 메뉴가 뜨는 구조로 간다. 메뉴는 "계속하기, 저장, 불러오기, 설정, 갤러리, 타이틀"이 한눈에 보이고, 텍스트 속도와 음량 조절이 실제 기능과 연결되어야 한다.

필요 에셋:
- 메뉴 패널 프레임.
- 메뉴 버튼 3상태.
- 닫기 아이콘.
- 볼륨 슬라이더.
- 텍스트 속도 슬라이더 또는 세그먼트 컨트롤.
- 미니 캐릭터 초상화 프레임.

## 구현 원칙

- 목업 이미지는 최종 UI의 방향을 정하는 참조로만 사용하고, 게임 화면 자체를 통짜 이미지로 쓰지 않는다.
- 사용자에게 보이는 텍스트는 전부 한국어로 통일한다.
- 캐릭터, UI 프레임, 버튼, 아이콘, 카드 프레임은 재사용 가능한 독립 에셋으로 분리한다.
- CSS/JS 구현은 목업 확정 이후에 시작한다.
- 미연시 화면은 캐릭터 레이어, 배경 레이어, 대화창 레이어, 선택지 레이어가 분리되어야 한다.
- README의 성적 콘텐츠 허용 기준을 따른다. 단, 모든 로맨스 대상은 성인으로 유지하고 UI/에셋은 설렘, 코미디, 상호 존중을 중심에 둔다.

## 다음 작업 순서

1. 목업 4종 중 최종 기준으로 삼을 화면을 고른다.
2. 캐릭터별 독립 일러스트를 먼저 생성한다.
3. 대화창, 버튼, 아이콘, 카드 프레임을 에셋 세트로 생성한다.
4. 기존 `src/styles.css`의 UI 레이아웃을 목업 기준으로 재작성한다.
5. `src/main.js`에 VN 대화 레이어, 튜토리얼 레이어, 전투 피드백 애니메이션 상태를 추가한다.
6. Playwright 스크린샷으로 데스크톱/모바일 화면을 비교 검수한다.
