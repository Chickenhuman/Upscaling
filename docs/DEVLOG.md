# 개발 일지

## 재개발 방향

중단된 작업 이후 새로 들어온 `네온 더비` 목업과 에셋을 기준으로 게임 구조를 재정렬했다. 기존 카드 결투 데이터와 새 경주 데이터가 충돌하던 상태를 해소하고, 미연시 대화, 전략 카드 덱, 레이스 결과, 보상, 멀티 엔딩이 하나의 루프로 이어지게 다시 구성했다.

## UI 우선 구현

- `assets/concepts`의 타이틀, 대화, 레이스, 베팅 목업을 화면 기준으로 삼았다.
- 통짜 목업 이미지를 배경으로 쓰지 않고, 생성 배경과 캐릭터/말 스프라이트를 분리해 실제 UI 레이어로 구현했다.
- VN 화면은 캐릭터 전신, 하단 대화창, 이름표, 선택지 스택을 분리했다.
- 레이스 화면은 말 스프라이트 시트와 CSS 애니메이션으로 순위 연출을 넣었다.

## 시스템 구현 기록

- `src/main.js`를 새 데이터 구조에 맞춰 재작성했다.
- `src/data/cards.js`에 20장의 전략 카드, 포커스 제한, 칩 목표, 밸런스 기준을 추가했다.
- `src/data/content.js`는 7장 스토리와 8종 엔딩을 유지하면서 검증용 스토리/대사 125페이지를 노출한다.
- `tools/smoke-test.mjs`는 새 에셋, 캐릭터, 말, 주로, 카드, 문서를 검사한다.
- `tools/balance-sim.mjs`는 5,000회 시뮬레이션으로 승률과 회차 길이를 출력하고 목표 승률을 벗어나면 실패한다.

## 밸런스 기록

현재 기준은 풀 클리어 목표 68분, 장별 레이스 제한 4회, 목표 승률 42-68%다. `npm run check`의 5,000회 시뮬레이션 결과 승률은 58.1%로 목표 범위 안에 있다.

## 남은 과제

- 실제 플레이 로그를 모아 카드별 선택률과 캐릭터 엔딩 도달률을 조정한다.
- 오디오가 추가될 경우 음량 설정과 별도 에셋 검증 항목을 확장한다.
- 모바일 실기기에서 캐릭터 스프라이트와 대화창 겹침을 추가 점검한다.
