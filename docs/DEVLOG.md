# 개발 일지

## 프로젝트 방향

바닐라 JavaScript만으로 가볍게 실행되는 웹 게임을 목표로 한다. 핵심 경험은 대화 선택, 덱 성장, 반복 플레이의 세 축이며, 플레이어가 매 회차 조금씩 더 많은 이야기와 전략을 발견하게 만드는 것이 중요하다.

## 설계 원칙

- 첫 화면은 바로 플레이 가능한 게임이어야 한다.
- UI는 짧은 세션과 반복 플레이에 맞게 빠르고 명확해야 한다.
- 로맨스는 캐릭터의 선택, 약점, 성장과 연결되어야 한다.
- 전투 보상은 덱 전략과 서사 해금을 동시에 자극해야 한다.
- 실패는 끝이 아니라 다음 회차의 정보가 되어야 한다.

## 콘텐츠 기준

- 15세 이상 이용 기준을 유지하되, 성적 콘텐츠 허용 수위와의 충돌 가능성을 출시 전 재검토한다.
- 모든 로맨스 가능 캐릭터는 성인으로 설정한다.
- 성인 캐릭터 간의 명시적 성적 콘텐츠는 일정 수위까지 허용한다.
- 선정적 표현이 들어가더라도 설렘, 코미디, 정서적 친밀감, 상호 존중을 우선한다.

## 개발 기록

### 초기 문서화

- 장르 혼합 방향을 비주얼 노벨, 덱빌딩, 로그라이크로 확정했다.
- 타이틀, ESC 메뉴, 덱, 맵, 전투, 엔딩 흐름을 문서화했다.
- 출시 전 검수에 필요한 안전 문구와 콘텐츠 기준을 명시했다.

### 시스템 우선순위

1. 타이틀, 저장, 설정, ESC 메뉴.
2. 대화 화면과 선택지 플래그.
3. 카드 데이터, 덱, 손패, 턴 처리.
4. 맵 노드와 보상 루프.
5. 캐릭터별 호감도와 엔딩 조건.
6. 밸런스, 접근성, 릴리스 검수.

### RC 1.0 구현 기록

- 바닐라 ES 모듈 기반의 정적 웹 앱으로 타이틀, 동행 선택, 대화, 맵, 전투, 보상, 덱, 갤러리, 설정, 엔딩 화면을 연결했다.
- 이매진 생성 에셋 6종을 프로젝트 내부 `assets/images`에 복사해 배경과 카드 아트로 사용했다.
- 스토리 125페이지, 대사 125페이지, 엔딩 8종, 카드 55장, 유물 12종, 12주 레벨 곡선을 런타임 데이터로 검증했다.
- 밸런스 시뮬레이션 5,000회 기준 승률 57.0%, 평균 회차 길이 71.7분, 풀 클리어 목표 78분으로 패치했다.
- `npm run check`와 브라우저 모듈 스모크, 로컬 HTTP 응답 확인을 완료했다.

## 남은 과제

- 실제 사용자 플레이 로그를 모아 캐릭터별 엔딩 도달률을 조정한다.
- Chrome 외 브라우저의 시각적 차이를 수동 확인한다.
- 향후 오디오를 추가할 경우 음량/음소거 저장 항목을 별도 체크한다.
